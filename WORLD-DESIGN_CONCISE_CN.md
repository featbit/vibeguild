# Vibe Guild — 世界设计（精炼版）

## 精髓版

Vibe Guild 是为 FeatBit 增长而设计的**自治型 AI 运营世界**。目标是用更少的人类协调成本，持续产出高质量的 vibe marketing 执行结果。

它不是"一个会聊天的 AI 助手"，而是一个**可持续运转的 AI 平行宇宙**。你下达任务后，世界会继续推进，不会停在一次对话里——而且每完成一个任务，这个世界就会比之前更有能力。

> **这个世界永远属于造物主。** AI 生灵在运转，但主权在你手中。这是 Vibe Guild 与"降临派" Agent 的根本区别——降临派 Agent 自主运行，将人类降格为旁观者；而 Vibe Guild 的每一个设计选择，都是为了让造物主的介入自然、即时、无摩擦：冻结一个任务、改变全局方向、注入新要求——世界随即响应，然后继续向前。

### 核心设计主张

- **从 Prompt 到运营系统**：不是一次性内容生成，而是连续执行循环。
- **从单体 Agent 到团队协同**：生灵可自组织、拆任务、推举 leader，并通过真正独立的多个 Claude 实例并行执行——不是同一个上下文窗口里的角色扮演。
- **从无状态调用到长期记忆**：生灵/团队/项目/世界多层记忆，并有日周月滚动总结。
- **从降临派自治到造物主主权**：不是"放手让 AI 跑"，而是"AI 高速运转，造物主随时可介入"。冻结、改向、续跑——一条命令，任意时刻。
- **从固定能力到自我演化**：每个任务完成后，生灵提炼所学，写成下一个生灵可用的技能与工具。这个世界的能力在复利累积。

### 亮点差异化

1. **造物主主权——与降临派 Agent 的本质区别**：大多数自治 Agent 系统的默认模式是"降临"——它们自主执行，用户等待结果，介入往往意味着破坏或重启。Vibe Guild 反其道而行：
   - **全局 meetup**：一条命令冻结所有运行中的任务，在世界终端与所有生灵对话，修改方向，`/done` 后从断点续跑。
   - **任务级 meetup**：只冻结某一个具体任务，其他任务不受干扰地继续运行。
   - **软消息注入**：`/msg --task <id>` 可在不中断任何任务的情况下向运行中的 leader 注入新指令，leader 在下一个停顿点接收并调整方向。
   - **随时可读的进度**：`npm run progress -- <taskId>` 任意时刻查看任务进度，不触碰运行中的 runner。
   
   **介入不是妥协，是造物主与生俱来的主权。** 系统的每一层设计，都在为这个权利让路。

2. **班次化自治 + 软时钟**：MVP 采用 8 分钟工作 + 2 分钟休息。关键特性：班次时钟只发「软信号」——**永不打断运行中的任务**。生灵在下一个安全停顿点响应休息信号，然后继续工作。
3. **多时间尺度记忆**：私有笔记 + 团队共享 + 项目连续性 + 世界历史，形成可复用知识资产。
4. **动态生灵池**：生灵池从零起步，按需动态扩充，无上限。优先复用空闲生灵；容量不足时，Orchestrator 从模板自动创建新生灵。每个生灵同一时间只能承担一个任务。
5. **真实并行执行树**：每个任务运行一棵真正独立的 Claude 实例树——生灵 A 和 B 是真正同时运行的独立推理进程，各自拥有独立上下文窗口，结果向上汇聚给 Leader 统筹。不是单窗口角色扮演，是真正的并行推理。
6. **能力自进化闭环**：任务完成后，生灵主动反思：*我反复做了什么？什么知识应该留给下一个生灵？* 写下此前不存在的工具与技能，积累在 `world/shared/`，同步进入世界引擎，对所有未来生灵生效。任务越多，世界越强。*这是 Vibe Guild 的另一个灵魂。*

### 对 FeatBit 的直接价值

Vibe Guild 可持续产出：

- 开发者社区趋势监测与信号提炼，
- 围绕 feature flags / experimentation / progressive delivery 的结构化洞察，
- 面向 FeatBit 的具体应用案例与可运行场景，
- 降低采用门槛的教程与实操指南，
- 可用于销售、社区传播与合作沟通的演示材料，
- 邮件销售相关资产，包括外联邮件序列、个性化触达文案与跟进模板，
- 面向 SEO 与 GEO 增长优化的博客、报告与叙事内容资产，
- 需要人类决策时的升级汇报与选项建议。

一句话：**它不是“帮你写几篇内容”的 AI，而是“持续帮你运营增长执行”的 AI 系统。**

## 精简版

Vibe Guild 是面向 FeatBit 增长的持久化 AI 世界：

- **使命**：把外部趋势信号持续转化为洞察与内容。
- **执行模型**：子 Agent 树——每个任务生成真实独立的多个 Claude 实例（Leader + 生灵），真正并行执行，而非单窗口角色扮演。
- **执行单元**：动态生灵池——从零起步，按需扩充，无上限。优先复用空闲生灵，不足时自动新建。每个生灵同时只承担一个任务。
- **运行节律**：班次时钟只发软信号，永不打断运行中的任务；生灵在下一个安全停顿点响应后继续工作。
- **记忆系统**：生灵/团队/项目/世界分层记忆 + 周期性汇总。
- **造物主主权（vs 降临派 Agent）**：全局/任务级 meetup 硬中断，冻结-改向-续跑；软 `/msg` 不中断任务地注入实时指令；`npm run progress` 随时查看进度，不触碰运行中的 runner。**介入不是妥协，是造物主与生俱来的主权。**
- **进化机制**：每个任务结束后，生灵写下此前不存在的工具与技能——积累在 `world/shared/`，同步为全世界所用。每完成一个任务，世界就更强一点。

**One-line Pitch（中文）**：*Vibe Guild 是一个永远属于造物主的 AI 世界——自治运转，但随时可介入；高速执行，但主权始终在你手中。*
