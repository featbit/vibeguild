# LP;HU — Low Polish, High Utility
### (formerly Vibe Guild)

> **LP;HU — Low Polish, High Utility**
>
> This is not an AI toy world.
> This is a practical, supervised, automation system for real FeatBit work.

---

## Mission

LP;HU is an automation-first, human-in-the-loop friendly agent system that helps FeatBit:

- trigger product demos from skills,
- validate skills through the demo-trigger flow,
- generate best-practice articles from real implementation workflows.

The goal is simple: shorten the path from skill trigger → demo execution → validation evidence → publishable guidance.

---

## What This Agent Is

LP;HU is a task execution framework with vertical capability focus.

It is designed to run practical work loops continuously while staying easy for the operator to steer:

1. receive a concrete FeatBit task,
2. execute with tools and skills,
3. checkpoint progress in real time,
4. pause for alignment when needed,
5. continue and finish with actionable outputs.

This system is intentionally built for usefulness, not architectural showmanship.

---

## Core Capabilities

### 1) Skill-to-Demo Execution

Use skills as triggers to produce FeatBit application demos quickly and repeatedly.

### 2) Skill Validation via Demo Flow

Validate skills through execution evidence in demo workflows, not by isolated authoring tasks.

### 3) Best-Practice Content Generation

Turn validated implementation flows into best-practice articles and operational playbooks.

---

## Core Technology Position

LP;HU keeps a stable foundation while adopting the latest model and agent capabilities.

- **Coding agent kernels:** Claude Code (`cc`), Codex, and Copilot.
- **Model strategy:** aggressively adopt latest LLM and agent methods (including agent teams) when they improve real outcomes.
- **Execution isolation:** Docker sandbox as the execution substrate for parallel programming and multi-task runs.
- **Operator UI surfaces:** terminal and Discord as the primary interaction interfaces.

No unnecessary abstractions. No speculative architecture layers.

---

## Human-in-the-Loop Is a First-Class Mechanism

LP;HU is built around the reality that drift happens in two places:

1. planning interpretation,
2. implementation path selection.

The fix is timely human correction, not blind autonomy.

So the system optimizes for:

- frequent progress visibility,
- low-friction pause/resume,
- explicit alignment checkpoints,
- fast continuation after operator feedback.

---

## Self-Iteration Strategy

LP;HU is designed to self-iterate in a practical sense:

- produce artifacts,
- verify artifacts,
- feed outcomes back into better skills and workflows,
- rerun with improved prompts, tools, and constraints.

This is controlled iteration, not unsupervised self-evolution.

---

## Non-Goals

- Building a fantasy AI society.
- Optimizing for polish over utility.
- Over-engineering around hypothetical future constraints.

If a change does not increase real FeatBit execution value now, it should not be added.

---

## Closing Principle

LP;HU means staying grounded:

- practical outputs,
- tight feedback loops,
- modern agent tooling,
- clear human control.

Low polish. High utility.
